# Bike Super Racing — Stage 2 Drawing Task for `map_01` Core Tiles

## 1. Назначение задачи
Документ фиксирует отдельное рабочее ТЗ на Stage 2 для `map_01` в current first playable Bike Super Racing.

Задача Stage 2:
- отрисовать базовый читаемый набор core tiles для `map_01`;
- получить production-ready основу для сборки первой трассы;
- подготовить базу для интеграции в Unity и проверки читаемости на gameplay scale.

Stage 2 не покрывает:
- расширенный декоративный набор;
- сложные фоновые системы;
- агрессивный параллакс;
- погодные варианты;
- пост-MVP разнообразие тайлов.

## 2. Состав Stage 2
В Stage 2 входят только следующие ассеты:
- `SPR_Map_Map01_Tile_Flat_A`;
- `SPR_Map_Map01_Tile_Ground_A`;
- `SPR_Map_Map01_Tile_Ground_B`;
- `SPR_Map_Map01_Tile_Edge_A`;
- `SPR_Map_Map01_Tile_Crest_A`;
- `SPR_Map_Map01_Tile_Slope_Up_Short`;
- `SPR_Map_Map01_Tile_Slope_Down_Short`;
- `SPR_Map_Map01_Tile_Ramp_Short`.

## 3. Главная задача Stage 2
Игрок должен за долю секунды понять:
- где активная поверхность трассы;
- где начинается наклон;
- где находится верхняя точка рельефа;
- где находится трамплин;
- как читается профиль дороги на скорости.

Если трасса красива, но не читается — Stage 2 провален.

## 4. Художественный target
`map_01` должна восприниматься как:
- чистая;
- спортивная;
- яркая;
- аркадная;
- честная по геометрии.

Она не должна восприниматься как:
- грязная и шумная;
- декоративно перегруженная;
- двусмысленная по форме поверхности;
- более важная, чем player object.

## 5. Размеры
### 5.1. Базовый тайл
- `16x16 px`.

### 5.2. Крупные элементы
- только кратные `16 px`.

### 5.3. Сетка
- работа строго в логике базовой сетки проекта;
- integer upscale only.

## 6. Surface readability rules
### 6.1. Что должно читаться сразу
Игрок должен мгновенно различать:
- ровную поверхность;
- подъём;
- спуск;
- гребень;
- трамплин.

### 6.2. Что обязательно сохранить
- чёткую верхнюю линию rideable surface;
- понятную массу грунта;
- низкий внутренний шум;
- читаемый контраст между поверхностью и фоном.

### 6.3. Что запрещено
Не допускается:
- рваная верхняя линия без gameplay-смысла;
- чрезмерная текстурность грунта;
- контурная грязь ради псевдореализма;
- двусмысленная форма трамплина.

## 7. Уровень детализации
Stage 2 должен использовать средне-низкую детализацию.

Приоритет такой:
1. silhouette of surface;
2. profile readability;
3. clean contrast;
4. only then minimal internal texture.

## 8. Цвет и материал
### 8.1. Основной материал
Для Stage 2 используется один базовый dirt-look material.

### 8.2. Требования
Необходимо обеспечить:
- активную поверхность, читаемую отдельно от нижней массы грунта;
- грунт темнее и плотнее фона;
- edge highlight, помогающий считывать рельеф;
- отсутствие слияния с тёмными колёсами `bike_01`.

## 9. Порядок исполнения
Строгий порядок:
1. `SPR_Map_Map01_Tile_Flat_A`;
2. `SPR_Map_Map01_Tile_Ground_A`;
3. `SPR_Map_Map01_Tile_Ground_B`;
4. `SPR_Map_Map01_Tile_Edge_A`;
5. `SPR_Map_Map01_Tile_Crest_A`;
6. `SPR_Map_Map01_Tile_Slope_Up_Short`;
7. `SPR_Map_Map01_Tile_Slope_Down_Short`;
8. `SPR_Map_Map01_Tile_Ramp_Short`;
9. собрать тестовую полосу трассы.

## 10. Промпт-ориентир для художника
Использовать следующий смысловой ориентир:

- pixel art dirt racing track tiles for side-view arcade motorcycle game, 16x16 readable surface geometry, low noise, clean silhouette, clear slope and ramp readability, browser-safe contrast, no decorative clutter.

## 11. Hand-off в Unity
На выходе Stage 2 должны быть переданы:
- PNG-файлы с каноническими именами;
- source file с аккуратной сеткой;
- краткая note по сборке, если тайлы требуют определённого порядка.

## 12. Definition of done
Stage 2 считается завершённым, если:
1. из набора можно собрать читаемый участок трассы;
2. flat / slope / crest / ramp считываются мгновенно;
3. профиль трассы понятен на уменьшенном масштабе;
4. тайлы не создают визуального хаоса;
5. naming соответствует канону.

## 13. Что проверять сразу после отрисовки
После первой отрисовки необходимо проверить:
- читается ли верхняя линия поверхности на уменьшенном масштабе;
- не слишком ли много шума внутри грунта;
- не теряется ли край трассы;
- не выглядит ли ramp слишком поздно читаемым;
- не сливаются ли тайлы между собой в одно грязное пятно.

## 14. Следующий шаг
После завершения Stage 2 необходимо:
- показать собранный strip mockup;
- проверить читаемость в связке с `bike_01`;
- только после этого переходить к finish set и background.
