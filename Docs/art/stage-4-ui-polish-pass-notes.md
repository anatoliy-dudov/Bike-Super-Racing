# Bike Super Racing — Stage 4 UI Polish Pass Notes

## Purpose
This note records the goals of the focused polish pass for the Stage 4 race UI must-have assets.

The polish pass stays intentionally narrow.
It improves only:
- timer panel readability and silhouette;
- result panel hierarchy;
- primary vs secondary button emphasis.

It does not expand shell scope.

## Why this pass exists
After the first Stage 4 draft, the UI direction was correct, but the following areas still had room for improvement:
- timer panel could feel cleaner and less placeholder-like;
- result panel could present current result and best result with stronger structure;
- button hierarchy could push restart more clearly above the secondary action.

## Goals of the polish pass
The polish pass must improve:
- timer panel confidence at browser scale;
- result panel current-time focus;
- best-time secondary readability;
- restart-first conversion hierarchy;
- overall racing-UI clarity without added noise.

The polish pass must preserve:
- low-noise art direction;
- compact HUD feel;
- countdown dominance only in its state;
- gameplay-first visual priority.

## What must not happen
The polish pass must not introduce:
- heavy decorative frames;
- oversized result panel structure;
- secondary button weight close to primary;
- shell-like complexity unrelated to the race loop.

## Review checklist
After this pass, review:
1. is the timer easier to parse instantly;
2. is the current result more clearly dominant on result screen;
3. is best time clearly secondary but still readable;
4. is restart more visually obvious than exit/back;
5. does the UI remain secondary to gameplay outside the result state.

## Intended decision after this pass
If the answers above are strong, the Stage 4 UI set should be accepted as the UI baseline for the current first playable milestone and the team should move to the VFX must-have set.
