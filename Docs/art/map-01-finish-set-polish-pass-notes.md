# Bike Super Racing — `map_01` Finish Set Polish Pass Notes

## Purpose
This note records the goals of the focused polish pass for the finish set of `map_01`.

The polish pass stays intentionally narrow.
It improves only:
- finish banner clarity;
- finish post proportions;
- finish readability against sky and track.

It does not expand the background scope.

## Why this pass exists
After the first Stage 3 draft, the finish zone already worked directionally, but the following areas still had room for improvement:
- banner pixel treatment could be cleaner;
- posts could feel slightly more intentional and less placeholder-like;
- finish composition could be tuned to stay readable without becoming visually heavy.

## Goals of the polish pass
The polish pass must improve:
- banner silhouette confidence;
- banner readability against the sky;
- post slimness and placement feeling;
- finish-zone clarity at gameplay scale.

The polish pass must preserve:
- low-noise background support;
- arcade simplicity;
- bike-first visual priority;
- fast finish readability.

## What must not happen
The polish pass must not introduce:
- heavy decoration;
- oversized finish architecture;
- thick posts that create a wall;
- more visual weight than the player bike.

## Review checklist
After this pass, review:
1. is the finish easier to identify instantly;
2. does the banner feel cleaner;
3. do the posts feel slimmer and more intentional;
4. does the bike still remain the main moving object near finish;
5. does the finish avoid becoming a visual obstacle.

## Intended decision after this pass
If the answers above are strong, the finish set should be accepted as the Stage 3 baseline and the team should move to race UI must-have assets.
